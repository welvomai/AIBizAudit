# Welvom Audit Service

FastAPI service that collects business assessment responses, generates AI-based audit reports, builds PDF output, and sends results by email.

## What This App Does

- Serves a questionnaire page at /questionnaire.
- Accepts submissions from:
  - /submit-assessment (web form flow)
  - /webhook/tally (Tally webhook flow)
- Saves lead data to Supabase.
- Runs AI analysis with Mistral.
- Generates a PDF report using Playwright + Jinja2 templates.
- Sends report emails and internal lead notifications via Resend.
- Uses a local product catalog file in data/ to enrich recommendations.

## Tech Stack

- Python 3.10+
- FastAPI + Uvicorn
- Jinja2 templates
- Playwright (Chromium) for PDF rendering
- pypdf (table of contents page detection)
- Supabase Python client
- Resend API
- Mistral API

## Project Structure

- app/main.py: FastAPI app entry point
- app/routes.py: API routes and background processing flow
- app/ai.py: AI analysis and business-case logic
- app/pdf.py: PDF rendering logic
- app/product_catalog.py: product catalog loading/matching
- app/database.py: Supabase persistence
- app/email_sender.py: email delivery
- app/templates/: questionnaire and report templates
- data/: local product catalog data

## Prerequisites

1. Python 3.10 or newer
2. pip
3. Playwright browser binaries (Chromium)

## Setup

1. Create and activate a virtual environment

Windows PowerShell:

python -m venv .venv
.\.venv\Scripts\Activate.ps1

2. Install dependencies

pip install -r requirements.txt

3. Install Playwright Chromium

playwright install chromium

4. Create a .env file in the repository root

Use this template (replace placeholders):

MISTRAL_API_KEY=your_mistral_api_key
RESEND_API_KEY=your_resend_api_key
SENDER_EMAIL=your_verified_sender_email
BOSS_EMAIL=internal_notification_email
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your_supabase_anon_or_service_key

5. Ensure a product catalog exists in data/

The app auto-discovers catalog files and supports .xlsx, .csv, and .json.

Recommended path:
- data/Welvom_AI_Productslist.xlsx

Optional explicit override:
- Set WELVOM_PRODUCT_CATALOG_PATH in .env to a specific file path.

## Run Locally

Start development server:

uvicorn app.main:app --reload

Open in browser:

http://127.0.0.1:8000/questionnaire

## Main Endpoints

- GET / -> redirects to /questionnaire
- GET /questionnaire -> assessment form page
- POST /submit-assessment -> web form submission
- POST /webhook/tally -> Tally webhook ingestion

## Notes

- Assessment processing is intentionally asynchronous/background so users get a fast submit response.
- If Supabase credentials are missing, the app continues but skips database writes.
- If email sending fails, processing continues and logs warnings.
- A background_task.log file may be generated by assessment background processing.

## Troubleshooting

- PDF errors about Playwright:
  - Re-run: playwright install chromium
- Missing DB writes:
  - Verify SUPABASE_URL and SUPABASE_KEY
- Email not sent:
  - Verify RESEND_API_KEY, SENDER_EMAIL, and BOSS_EMAIL
- AI analysis not generated:
  - Verify MISTRAL_API_KEY

## Security

- Do not commit .env or API keys.
- Rotate keys immediately if they are accidentally exposed.
